# Auteo 桌面应用入口规格

## 组件职责

`src/main.rs` 是 Auteo 桌面应用入口，同时定义根级 `AuteoApp` 视图。它负责初始化 GPUI 与 `gpui-component`、创建主窗口、配置透明标题栏，并呈现 AI 剪辑师产品的品牌化起始界面。

## 使用场景

- 作为 Auteo macOS 应用安装、启动和品牌展示的最小可运行入口。
- 验证 GPUI 与 `gpui-component` 的根视图、主题背景和主按钮组合。
- 为后续素材导入、时间线编辑和 AI 剪辑工作流提供原生桌面基础。

## 数据输入与状态

- 组件没有外部 props 或运行时数据输入。
- `started` 是组件内部布尔状态，初始值为 `false`。
- 用户激活主按钮后，`started` 变为 `true`，并触发视图重绘。

## 视觉状态

- 主内容在窗口内水平和垂直居中，标题、说明文案和按钮纵向排列。
- 背景色、前景色和按钮样式由 `gpui-component` 当前主题提供，不在业务视图中硬编码。
- macOS 标题栏使用透明外观，内容背景延伸至窗口顶部，同时保留系统交通灯按钮。
- 初始状态显示 Auteo 品牌名、AI 视频剪辑定位文案和“Get started”按钮。
- 激活状态显示准备就绪文案和“Ready”按钮标签。

## 交互状态

- 点击或通过键盘激活主按钮后进入准备就绪状态。
- 重复激活按钮保持准备就绪状态，不产生额外副作用。
- hover、focus 和按压反馈沿用 `gpui-component` 主按钮的默认行为。
- 当前起始界面没有 disabled 或 loading 状态。

## 键盘与可访问性

- 主按钮必须可以通过标准键盘焦点顺序访问和激活。
- 焦点轮廓、键盘行为和语义由 `gpui-component` 按钮组件提供。
- 所有可见文案使用英文，并保持简短、明确。

## 边界状态

- 应用没有加载、空态、错误态、权限态、筛选、排序或分页行为。
- 主窗口创建失败时，应用通过明确的错误信息终止。
- 窗口尺寸变化时，内容持续保持居中，不使用固定内容尺寸。

## 组合关系

- `Application` 管理应用生命周期。
- `gpui_component::init` 注册主题和组件基础能力。
- `Root` 提供主题背景、默认字体、前景色以及弹层基础设施。
- `AuteoApp` 作为 `Root` 的内容视图。
- `Button` 提供唯一的用户交互入口。

## 已知限制与非目标

- 当前版本不包含素材导入、持久化、网络请求、时间线、AI 剪辑、路由、多窗口或复杂表单。
- 当前版本不自定义主题，仅使用品牌文案验证默认主题表现。
- 透明标题栏重点验证 macOS 表现；其它平台沿用 GPUI 的平台实现。
